## 3. `compose.yaml` Design

### 3.1 Key decisions

**Network:** The network is declared as an internal bridge named `fawkes-backbone-net`
within the compose project `ufawkes-resources`. This produces the full external name
`ufawkes-resources_fawkes-backbone-net` that downstream planes reference. The network
is created by Docker Compose on `make up` — it is **not** pre-created with
`docker network create` (unlike all other planes, which use `external: true`).

**Traefik Docker socket:** Traefik mounts `/var/run/docker.sock` read-only to discover
container labels. It must also be on `fawkes-backbone-net` so it can proxy to containers
on that network.

**Authelia `depends_on`:** Must wait for both `fawkes-postgres` and `fawkes-cache` to
be healthy before starting. Uses `condition: service_healthy`.

**Postgres port exposure:** Port 5432 exposed to host for dev convenience. This must
be documented as a dev-only setting. In production, remove the `ports:` block.

**Valkey authentication:** Valkey 8.x supports `requirepass` by default.
Use `command: valkey-server --requirepass ${VALKEY_PASSWORD}` and inject the password
via `.env`. All downstream connection strings must include the password.
Do not run Valkey without authentication even in local dev.

### 3.2 Target `compose.yaml`

```yaml
# uFawkesRes compose.yaml — v0.2
# This is the foundation plane. Start it first, shut it down last.
# Run: make init && make up
# After first boot: make db-create

services:
  traefik:
    image: traefik:v3.7.5
    container_name: traefik
    restart: unless-stopped
    command:
      - "--configFile=/etc/traefik/traefik.yml"
    ports:
      - "80:80"
      - "8080:8080" # Traefik dashboard (dev only)
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - ./traefik/traefik.yml:/etc/traefik/traefik.yml:ro
      - ./traefik/dynamic:/etc/traefik/dynamic:ro
    networks:
      - fawkes-backbone-net
    healthcheck:
      test: ["CMD", "traefik", "healthcheck", "--ping"]
      interval: 10s
      timeout: 5s
      retries: 3

  fawkes-sso:
    image: authelia/authelia:4.38.17 # VERIFY tag at github.com/authelia/authelia/releases
    container_name: fawkes-sso
    restart: unless-stopped
    depends_on:
      fawkes-postgres:
        condition: service_healthy
      fawkes-cache:
        condition: service_healthy
    volumes:
      - ./authelia:/config
    secrets:
      - authelia_jwt_secret
      - authelia_session_secret
      - authelia_storage_password
      - authelia_storage_encryption_key
    environment:
      AUTHELIA_IDENTITY_VALIDATION_RESET_PASSWORD_JWT_SECRET_FILE: /run/secrets/authelia_jwt_secret
      AUTHELIA_SESSION_SECRET_FILE: /run/secrets/authelia_session_secret
      AUTHELIA_STORAGE_POSTGRES_PASSWORD_FILE: /run/secrets/authelia_storage_password
      AUTHELIA_STORAGE_ENCRYPTION_KEY_FILE: /run/secrets/authelia_storage_encryption_key
    ports:
      - "9091:9091"
    networks:
      - fawkes-backbone-net
    labels:
      traefik.enable: "true"
      # Authelia itself is routed by Traefik
      traefik.http.routers.authelia.rule: "Host(`auth.localhost`)"
      traefik.http.routers.authelia.entrypoints: "web"
      traefik.http.services.authelia.loadbalancer.server.port: "9091"
      # ForwardAuth middleware — referenced by ALL downstream services
      traefik.http.middlewares.authelia.forwardauth.address: "http://fawkes-sso:9091/api/authz/forward-auth"
      traefik.http.middlewares.authelia.forwardauth.trustForwardHeader: "true"
      traefik.http.middlewares.authelia.forwardauth.authResponseHeaders: "Remote-User,Remote-Groups,Remote-Name,Remote-Email"
    healthcheck:
      test:
        ["CMD", "wget", "-q", "--spider", "http://localhost:9091/api/health"]
      interval: 15s
      timeout: 5s
      retries: 5
      start_period: 10s

  fawkes-postgres:
    image: postgres:17-alpine
    container_name: fawkes-postgres
    restart: unless-stopped
    environment:
      POSTGRES_USER: ${POSTGRES_SUPERUSER:-postgres}
      POSTGRES_PASSWORD_FILE: /run/secrets/postgres_superuser_password
      POSTGRES_DB: postgres
    secrets:
      - postgres_superuser_password
    volumes:
      - postgres-data:/var/lib/postgresql/data
    ports:
      - "5432:5432" # dev only — remove in production
    networks:
      - fawkes-backbone-net
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_SUPERUSER:-postgres}"]
      interval: 5s
      timeout: 5s
      retries: 10
      start_period: 10s

  fawkes-cache:
    image: valkey/valkey:8.1-alpine
    container_name: fawkes-cache
    restart: unless-stopped
    command: valkey-server --requirepass ${VALKEY_PASSWORD} --appendonly yes
    environment:
      VALKEY_PASSWORD: ${VALKEY_PASSWORD}
    volumes:
      - valkey-data:/data
    ports:
      - "6379:6379" # dev only — remove in production
    networks:
      - fawkes-backbone-net
    healthcheck:
      test: ["CMD", "valkey-cli", "-a", "${VALKEY_PASSWORD}", "ping"]
      interval: 5s
      timeout: 5s
      retries: 5

secrets:
  authelia_jwt_secret:
    file: ./authelia/secrets/JWT_SECRET
  authelia_session_secret:
    file: ./authelia/secrets/SESSION_SECRET
  authelia_storage_password:
    file: ./authelia/secrets/STORAGE_PASSWORD
  authelia_storage_encryption_key:
    file: ./authelia/secrets/STORAGE_ENCRYPTION_KEY
  postgres_superuser_password:
    file: ./authelia/secrets/POSTGRES_SUPERUSER_PASSWORD

volumes:
  postgres-data:
  valkey-data:

networks:
  fawkes-backbone-net:
    driver: bridge
    # Note: this is NOT external — uFawkesRes OWNS this network.
    # Downstream planes attach to it as external with:
    #   name: ufawkes-resources_fawkes-backbone-net
    #   external: true
```

**CAUTION on Authelia ForwardAuth middleware label:** The `forwardauth.address` value
`http://fawkes-sso:9091/api/authz/forward-auth` is the Authelia v4.38+ endpoint path.
This path changed in Authelia v4.37. If your Authelia version differs, verify the
correct path at https://www.authelia.com/integration/proxies/traefik/ before
implementing RES-002.

**CAUTION on Valkey password in healthcheck:** `valkey-cli -a ${VALKEY_PASSWORD} ping`
exposes the password in `ps` output. For v0.3, use a secrets file instead.
Acceptable for local dev.

```yaml
  dashboard: true
  insecure: true # Dashboard on :8080 without auth — dev only

  docker:
    exposedByDefault: false # Only route containers with traefik.enable=true

  file:
    directory: /etc/traefik/dynamic
    watch: true # Hot-reload dynamic config changes

  web:
    address: ":80"
    # forwardedHeaders for Authelia — trust X-Forwarded-* from loopback

      trustedIPs:
        - "127.0.0.1/32"
        - "172.16.0.0/12" # Docker network range

log:
  level: INFO

accessLog: {}
```

## 6. `authelia/configuration.yml` Design

This is the minimum viable Authelia configuration for local dev. Do not invent field
names — all fields below are from the Authelia v4.38 documentation.

```yaml
# Authelia configuration — v0.2 local dev baseline
# VERIFY all field names against https://www.authelia.com/configuration/

theme: light
default_redirection_url: http://localhost/

server:
  host: 0.0.0.0
  port: 9091

log:
  level: info

totp:
  issuer: fawkes-idp

authentication_backend:
  file:
    path: /config/users_database.yml
    password:
      algorithm: argon2id

access_control:
  default_policy: deny
  rules:
    # Traefik dashboard — platform engineers only
    - domain: traefik.localhost
      policy: one_factor
    # All other services require authentication
    - domain: "*.localhost"
      policy: one_factor

session:
  name: authelia_session
  expiration: 3600
  inactivity: 300
  remember_me_duration: 1M
  domain: localhost
  redis:
    host: fawkes-cache
    port: 6379
    password: ${VALKEY_PASSWORD} # VERIFY: Authelia supports env var here
    database_index: 0

storage:
  postgres:
    host: fawkes-postgres
    port: 5432
    database: authelia
    username: authelia
    # password from Docker secret via AUTHELIA_STORAGE_POSTGRES_PASSWORD_FILE

notifier:
  # No SMTP for local dev — use filesystem notifier instead
  filesystem:
    filename: /config/notification.txt
```

**CAUTION:** The `${VALKEY_PASSWORD}` interpolation in `configuration.yml` — verify
whether Authelia supports environment variable substitution in config files at your
target version. If not, use the `AUTHELIA_SESSION_REDIS_PASSWORD_FILE` env var instead.

## 8. `scripts/init.sh` Design

`make init` calls this script. It must be idempotent — running it twice on an existing
install does not overwrite existing secrets.

```bash
#!/usr/bin/env bash
set -euo pipefail

mkdir -p authelia/secrets

generate_secret() {
  local path="$1"
  local length="${2:-64}"
  if [ ! -f "$path" ]; then
    openssl rand -base64 "$length" | tr -d '\n' > "$path"
    echo "Generated: $path"
  else
    echo "Exists (skipped): $path"
  fi
}

generate_secret authelia/secrets/JWT_SECRET
generate_secret authelia/secrets/SESSION_SECRET
generate_secret authelia/secrets/STORAGE_PASSWORD 32
generate_secret authelia/secrets/STORAGE_ENCRYPTION_KEY 32
generate_secret authelia/secrets/POSTGRES_SUPERUSER_PASSWORD 32

echo ""
echo "Secrets generated. Next steps:"
echo "1. Generate an Authelia password hash:"
echo "   docker run --rm authelia/authelia:4.38.17 authelia crypto hash generate argon2id"
echo "2. Replace the placeholder hash in authelia/users_database.yml"
echo "3. Copy .env.example to .env and fill in VALKEY_PASSWORD and other vars"
echo "4. Run: make up"
echo "5. After first boot: make db-create"
```

## 9. `scripts/db-create.sh` Design

Creates per-plane databases and users. Run once after first `make up`.

```bash
#!/usr/bin/env bash
set -euo pipefail

# Reads superuser password from secret file
PG_PASS=$(cat authelia/secrets/POSTGRES_SUPERUSER_PASSWORD)

run_sql() {
  docker exec -i fawkes-postgres psql \
    -U "${POSTGRES_SUPERUSER:-postgres}" \
    -c "$1"
}

create_db_and_user() {
  local db="$1"
  local user="$2"
  local pass="$3"
  echo "Creating database '$db' and user '$user'..."
  run_sql "CREATE USER $user WITH PASSWORD '$pass';" 2>/dev/null || echo "  User '$user' already exists"
  run_sql "CREATE DATABASE $db OWNER $user;" 2>/dev/null || echo "  Database '$db' already exists"
  run_sql "GRANT ALL PRIVILEGES ON DATABASE $db TO $user;"
}

# Authelia — must already exist (created by make init separately)
# create_db_and_user "authelia" "authelia" "$(cat authelia/secrets/STORAGE_PASSWORD)"

# uFawkesDevX
create_db_and_user "coder"      "coder"      "${CODER_DB_PASSWORD:?set in .env}"
create_db_and_user "backstage"  "backstage"  "${BACKSTAGE_DB_PASSWORD:?set in .env}"

# uFawkesSec
create_db_and_user "dojo"       "dojo"       "${DOJO_DB_PASSWORD:?set in .env}"
create_db_and_user "infisical"  "infisical"  "${INFISICAL_DB_PASSWORD:?set in .env}"

echo ""
echo "Databases created. Add the passwords above to the relevant plane .env files."
```

## 10. Makefile Design

```makefile
.PHONY: init up down db-create logs-traefik logs-authelia test help

init: ## Generate secrets, create config dirs (idempotent — safe to run again)
 @bash scripts/init.sh

up: ## Start uFawkesRes stack (run make init first)
 docker compose up -d
 @echo ""
 @echo "  Traefik gateway:   http://localhost:80"
 @echo "  Traefik dashboard: http://localhost:8080"
 @echo "  Authelia SSO:      http://localhost:9091"
 @echo ""
 @echo "Run 'make db-create' after first boot to provision per-plane databases."

down: ## Stop stack (keep volumes — data is preserved)
 docker compose down

down-volumes: ## Stop stack AND delete all data volumes
 docker compose down -v

db-create: ## Create per-plane databases in Postgres (run once after first up)
 @bash scripts/db-create.sh

logs-traefik: ## Tail Traefik logs
 docker compose logs -f traefik

logs-authelia: ## Tail Authelia logs
 docker compose logs -f fawkes-sso

test: ## Run structural contract tests
 pytest tests/unit/ -v

pre-commit-setup: ## Install pre-commit hooks
 pip install pre-commit && pre-commit install

pre-commit-run: ## Run pre-commit on all files
 pre-commit run --all-files

help:
 @grep -E '^[a-zA-Z_-]+:.*?## .*$$' $(MAKEFILE_LIST) | \
   awk 'BEGIN {FS = ":.*?## "}; {printf "  %-20s %s\n", $$1, $$2}'
```

## 12. Risks and Mitigations

| Risk                                                                                                       | Likelihood                                           | Mitigation                                                                                                        |
| ---------------------------------------------------------------------------------------------------------- | ---------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| Authelia fails to start because `authelia` DB does not exist                                               | **High** (bootstrap ordering problem)                | `make init` creates the `authelia` DB using `psql` directly before starting the stack; document in quickstart     |
| `${VALKEY_PASSWORD}` in Valkey healthcheck exposed in process list                                         | Medium                                               | Acceptable for v0.2 dev; note in code comment; fix in v0.3 using secrets file                                     |
| Authelia ForwardAuth middleware path incorrect for chosen version                                          | Medium                                               | Pin Authelia version; add `test_authelia_config.py` assertion; verify path at docs before RES-002                 |
| Traefik Docker socket mount creates security surface                                                       | Always present                                       | Mount as `:ro`; this is the standard pattern; document explicitly                                                 |
| Downstream plane uses wrong network name (`fawkes-net` instead of `ufawkes-resources_fawkes-backbone-net`) | **High** (all prior docs in session used wrong name) | RES-004 produces a `docs/connecting-downstream.md` with the exact YAML block to copy; amend fawkes-integration.md |
| `make down-volumes` accidentally destroys Postgres data                                                    | Medium                                               | Separate `down` (safe) and `down-volumes` (destructive) targets with distinct names; no `-v` flag on `down`       |
